What we've arrived at

The new source organization depends on a three pronged attack:

  • RCS to cache all the revisions to an single file
  • rcsvg to extract stable versions of each product from its own source cache under /usr/msrc/dept
  • packages to group related products and documents into releases

In addition the build environment is reused from the older master source system to provide:

  • platform indemendent builds
  • multiple developers working in parallel
  • a single source cache for any given product
  • source reuse and sharing among departments
